# Security Policy

## Reporting a Vulnerability

Please report security vulnerabilities to **security@groundcover.com**. Do not open a public GitHub issue for security concerns.

Include as much detail as you can: a description of the vulnerability, steps to reproduce it, and the potential impact. We'll acknowledge your report within 48 hours and keep you updated as we investigate and address the issue.

We ask that you give us reasonable time to fix the problem before any public disclosure.

## Supported Versions

| Version | Supported |
| ------- | --------- |
| v1.x    | Yes       |

Older major versions are not actively maintained. If you're on an older version, upgrade to v1.x before reporting.

## Scope

This policy covers the action source code in this repository. It does not cover third-party OTLP backends or the GitHub Actions platform itself.
